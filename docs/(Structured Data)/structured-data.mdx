The **Structured Data** layer of MDX provides semantic frontmatter using YAML-LD (YAML + Linked Data), enabling type-safe schemas, validation, and database storage.

## Core Concepts

### [Document](/Document)

The fundamental unit - combines YAML-LD frontmatter with content body.

```yaml
---
$type: BlogPost
$id: https://example.com/posts/hello
title: Hello World
author: https://example.com/authors/jane
---
```

### [Database](/Database)

Schema-first database modeling with automatic bi-directional relationships.

```typescript
const db = DB({
  Post: {
    title: 'string',
    author: 'Author.posts',  // Creates both directions
  },
})
```

### Templates (Shapes)

Templates define the **shape** of your data - the TypeScript type or JSON Schema that structures your content:

```typescript
type StoryBrand = {
  name: string
  hero: {
    persona: string
    occupation: string
  }
  problem: {
    external: string
    internal: string
  }
}

type Noun = {
  name: string
  plural: string
  properties: Array<{ name: string; type: string; required: boolean }>
  actions: Array<{ name: string; returns: string }>
}
```

Templates enable **bi-directional conversion** between objects and content formats:

```typescript
import { toMarkdown, fromMarkdown } from '@mdxld/markdown'
import { toJSONLD, toJSONSchema } from '@mdxld/json'

// Object → Formats
const markdown = toMarkdown(storyBrand)
const jsonld = toJSONLD(storyBrand)
const schema = toJSONSchema(StoryBrandType)

// Markdown → Object
const data = fromMarkdown<StoryBrand>(editedMarkdown)
```

See [Unstructured Content](/unstructured-content) for markdown layouts and [UI Components](/ui-components) for JSX rendering.

## Packages

### [mdxld](/mdxld)

Parse, stringify, validate, and compile MDXLD documents.

```bash
npm install mdxld
```

### [mdxdb](/mdxdb)

Database adapters for storing MDX documents:

- **@mdxdb/fs** - Filesystem (git-friendly)
- **@mdxdb/sqlite** - SQLite with vector search
- **@mdxdb/postgres** - PostgreSQL with pgvector
- **@mdxdb/mongo** - MongoDB with Atlas Vector Search
- **@mdxdb/clickhouse** - ClickHouse for analytics
