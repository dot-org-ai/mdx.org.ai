# BuilderCSS

A pure semantic HTML/CSS design system. No utility classes. No data attributes. Just meaningful HTML structure and CSS pattern-matching.

## Philosophy

BuilderCSS styles HTML based on **semantic structure**, not arbitrary class names or data attributes. The HTML itself declares intent—CSS uses `:has()` selectors to pattern-match that intent.

```html
<!-- The structure IS the styling API -->
<article>
  <aside>
    <nav>...</nav>
  </aside>
  <main>...</main>
</article>
```

```css
/* CSS pattern-matches semantic structure */
article:has(> aside:first-child > nav) {
  display: grid;
  grid-template-columns: 280px 1fr;
}
```

## How It Works

Components are identified by their semantic HTML structure:

| Pattern | CSS Selector | Meaning |
|---------|-------------|---------|
| `article > aside + main` | `article:has(> aside:first-child)` | Sidebar layout |
| `section > h1:first-child` | `section:has(> h1:first-child)` | Hero section |
| `section > dl` | `section:has(> dl)` | FAQ section |
| `section > blockquote` | `section:has(> blockquote)` | Testimonials |
| `dialog` | `dialog` | Modal |
| `article > table` | `article:has(> table)` | Table view |

## Quick Example

```html
<article>
  <header>
    <nav>
      <a href="/">BuilderCSS</a>
      <menu>
        <li><a href="/docs">Docs</a></li>
        <li><a href="/pricing">Pricing</a></li>
      </menu>
    </nav>
  </header>
  <main>
    <section>
      <h1>Build semantic interfaces</h1>
      <p>Pure HTML/CSS that works everywhere.</p>
      <menu>
        <li><a href="/quickstart">Get Started</a></li>
        <li><a href="/demo">View Demo</a></li>
      </menu>
    </section>
  </main>
  <footer>
    <p>&copy; 2024 BuilderCSS</p>
  </footer>
</article>
```

```css
/* Stacked layout = article with header + main + footer */
article:has(> header + main + footer) {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}

/* Hero = first section with h1 */
section:has(> h1:first-child) {
  text-align: center;
  padding: var(--spacing-16) var(--spacing-4);
}

/* Primary action = first item in menu */
section:has(> h1) menu > :first-child a {
  background: var(--primary);
  color: var(--primary-foreground);
  padding: var(--spacing-3) var(--spacing-6);
  border-radius: var(--radius);
}
```

## States via ARIA

Interactive states use accessibility attributes—not data attributes:

```html
<nav>
  <a href="/" aria-current="page">Home</a>
  <a href="/about">About</a>
</nav>

<button aria-pressed="true">Toggle</button>

<details open>
  <summary>Expand</summary>
  <p>Content</p>
</details>
```

```css
[aria-current="page"] {
  background: var(--primary);
  color: var(--primary-foreground);
}

[aria-pressed="true"] {
  background: var(--muted);
}

details[open] > summary {
  font-weight: 600;
}
```

## Design Tokens

All visual properties use CSS custom properties:

```css
:root {
  /* Colors */
  --background: 0 0% 100%;
  --foreground: 222.2 84% 4.9%;
  --primary: 222.2 47.4% 11.2%;
  --primary-foreground: 210 40% 98%;

  /* Spacing */
  --spacing-1: 0.25rem;
  --spacing-2: 0.5rem;
  --spacing-4: 1rem;
  --spacing-8: 2rem;

  /* Radii */
  --radius: 0.375rem;
}
```
