# BuilderCSS

A semantic HTML/CSS design system for building AI-native applications across web and mobile platforms.

## Philosophy

BuilderCSS embraces pure semantic HTML with CSS selectors—no utility classes, no JavaScript frameworks required. Components are defined using data attributes and semantic elements that work natively in browsers and translate cleanly to React Native for mobile.

## Quick Example

```html
<!-- Semantic structure -->
<body data-layout="stacked">
  <header>
    <nav>
      <a href="/">BuilderCSS</a>
      <ul>
        <li><a href="/docs">Docs</a></li>
        <li><a href="/pricing">Pricing</a></li>
      </ul>
    </nav>
  </header>
  <main>
    <section data-section="hero">
      <h1>Build semantic interfaces</h1>
      <p>Pure HTML/CSS that works everywhere.</p>
      <a href="/quickstart" data-button="primary">Get Started</a>
    </section>
  </main>
  <footer>
    <p>&copy; 2024 BuilderCSS</p>
  </footer>
</body>
```

```css
/* Pure CSS styling */
[data-layout="stacked"] {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}

[data-section="hero"] {
  text-align: center;
  padding: var(--spacing-16) var(--spacing-4);
}

[data-button="primary"] {
  display: inline-flex;
  padding: var(--spacing-3) var(--spacing-6);
  background: var(--primary);
  color: var(--primary-foreground);
  border-radius: var(--radius-md);
}
```

## Platforms

BuilderCSS provides implementations for:

- **Web** - Pure HTML/CSS with semantic selectors
- **React** - Component wrappers with TypeScript
- **React Native** - Expo-compatible mobile components

## Design Tokens

All components use CSS custom properties for theming:

```css
:root {
  /* Colors */
  --background: 0 0% 100%;
  --foreground: 222.2 84% 4.9%;
  --primary: 222.2 47.4% 11.2%;
  --primary-foreground: 210 40% 98%;

  /* Spacing */
  --spacing-1: 0.25rem;
  --spacing-2: 0.5rem;
  --spacing-4: 1rem;
  --spacing-8: 2rem;

  /* Radii */
  --radius-sm: 0.25rem;
  --radius-md: 0.375rem;
  --radius-lg: 0.5rem;
}
```
