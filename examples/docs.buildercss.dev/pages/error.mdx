# Error Page

A generic error page for server errors and unexpected issues.

<Preview>
  <main>
    <figure aria-hidden="true">!</figure>
    <h1>Something went wrong</h1>
    <p>We're sorry, but something unexpected happened. Please try again later or contact support if the problem persists.</p>

    <menu>
      <li><button>Try Again</button></li>
      <li><a href="#">Go Home</a></li>

    <details>
      <summary>Technical details</summary>
      <pre>
Error Code: 500
Request ID: abc-123-def-456
Timestamp: 2024-12-01T10:30:00Z
      </pre>
    </details>
  </main>
</Preview>

## Semantic Pattern

An error page uses `<main>` with an error icon figure, heading, message, actions, and collapsible technical details. The pattern `main:has(> figure[aria-hidden] + h1 + p + menu)` identifies error pages:

```html
<main>
  <figure aria-hidden="true">!</figure>
  <h1>Something went wrong</h1>
  <p>We're sorry, but something unexpected happened.</p>

  <menu>
    <li><button onclick="location.reload()">Try Again</button></li>
    <li><a href="/">Go Home</a></li>
  </menu>

  <details>
    <summary>Technical details</summary>
    <pre>
Error Code: 500
Request ID: abc-123-def-456
Timestamp: 2024-12-01T10:30:00Z
    </pre>
  </details>
</main>
```

## When to Use

- 500 server error pages
- Application crash states
- Unexpected error handling
- Service unavailable pages
