## Semantic Pattern

A waitlist page uses `<main>` with a badge, heading, email form, counter, and perks section. The pattern `main:has(> form + p[aria-label="Count"])` identifies waitlist pages:

```html
<main>
  <mark>Coming Soon</mark>
  <h1>Be the first to experience Builder CSS 3.0</h1>
  <p>Join the waitlist to get early access...</p>

  <form>

    <button type="submit">Join Waitlist</button>
  </form>
```

## JavaScript

```js
const form = document.querySelector('main form')
const countEl = document.querySelector('main data')

form.addEventListener('submit', async (e) => {
  e.preventDefault()
  const email = form.querySelector('input').value

  // Submit to API
  const res = await fetch('/api/waitlist', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ email })
  })

  if (res.ok) {
    // Update count and show success
    const { count } = await res.json()
    countEl.textContent = count.toLocaleString()
    form.innerHTML = '<p>ğŸ‰ You\'re on the list!</p>'
  }
})
```
