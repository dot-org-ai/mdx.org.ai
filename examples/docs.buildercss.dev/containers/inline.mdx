# Inline Container

Content embedded directly within the page flow, always visible without overlays.

<Preview>
  <div>

    <aside>
      <header>

      </header>
      <p>Experience our app in a whole new way with our brand new dark mode feature.</p>

        <li><button>Try it now</button></li>
        <li><a href="#">Learn more</a></li>

    </aside>

    <p>More content continues below...</p>
  </div>
</Preview>

## Semantic Pattern

An inline container is an `<aside>` with announcement/card content in the page flow. The pattern `aside:has(> header > mark)` identifies feature announcements:

```html
<article>
  <p>Regular content...</p>

  <aside>
    <header>
      <mark>NEW</mark>
      <h3>Feature Announcement</h3>
    </header>
    <p>Description of the feature...</p>
    <menu>
      <li><a href="/try">Try it now</a></li>
      <li><a href="/learn">Learn more</a></li>
    </menu>
  </aside>

  <p>More content...</p>
</article>
```

## CSS

```css
/* Inline card: aside with header containing mark */
aside:has(> header > mark) {
  background: var(--muted);
  border: 1px solid var(--border);
  border-radius: var(--radius-lg);
  padding: var(--spacing-6);
  margin: var(--spacing-6) 0;
}

aside:has(> header > mark) > header {
  margin-bottom: var(--spacing-4);
}

aside:has(> header > mark) > header > mark {
  display: inline-block;
  padding: var(--spacing-1) var(--spacing-2);
  background: var(--primary);
  color: var(--primary-foreground);
  border-radius: var(--radius);
  font-size: var(--text-xs);
  margin-bottom: var(--spacing-2);
}

aside:has(> header > mark) > header h3 {
  font-size: var(--text-lg);
  font-weight: 600;
}

aside:has(> header > mark) > p {
  color: var(--muted-foreground);
  margin-bottom: var(--spacing-4);
}

aside:has(> header > mark) > menu {
  display: flex;
  gap: var(--spacing-2);
  list-style: none;
}

/* Primary: first action */
aside:has(> header > mark) > menu > :first-child button,
aside:has(> header > mark) > menu > :first-child a {
  display: inline-block;
  padding: var(--spacing-2) var(--spacing-4);
  background: var(--primary);
  color: var(--primary-foreground);
  border: none;
  border-radius: var(--radius);
  cursor: pointer;
  text-decoration: none;
}

/* Secondary: other actions */
aside:has(> header > mark) > menu > :not(:first-child) a {
  display: inline-block;
  padding: var(--spacing-2) var(--spacing-4);
  background: transparent;
  border: 1px solid var(--border);
  border-radius: var(--radius);
  cursor: pointer;
  text-decoration: none;
}
```

## Variations

Use `<aside>` with a figure for icons:

```html

  <div>

    <p>You can use keyboard shortcuts to speed up your workflow.</p>

```

```css
aside:has(> figure + div) {
  display: flex;
  gap: var(--spacing-3);
  padding: var(--spacing-4);
  background: var(--accent);
  border-left: 4px solid var(--accent-foreground);
  border-radius: var(--radius);
  margin: var(--spacing-4) 0;
}

aside:has(> figure + div) > figure {
  margin: 0;
  font-size: var(--text-xl);
}
```

### Dismissible Banner

```html
<aside>
  <p>üéâ New feature available! Check out our latest update.</p>
  <button aria-label="Dismiss">&times;</button>
</aside>
```

```css
aside:has(> p + button[aria-label="Dismiss"]) {
  display: flex;
  align-items: center;
  justify-content: space-between;
  padding: var(--spacing-3) var(--spacing-4);
  background: var(--primary);
  color: var(--primary-foreground);
  border-radius: var(--radius);
  margin: var(--spacing-4) 0;
}

aside:has(> p + button[aria-label="Dismiss"]) button {
  background: none;
  border: none;
  color: inherit;
  cursor: pointer;
  font-size: var(--text-lg);
}
```

```js
document.querySelectorAll('aside button[aria-label="Dismiss"]').forEach(btn => {
  btn.addEventListener('click', () => {
    const container = btn.closest('aside')
    container.remove()
    // Optionally persist dismissal
    localStorage.setItem(`dismissed-${container.id}`, 'true')
  })
})
```

### Collapsible Details

Use native `<details>` for collapsible content:

```html
<details>
  <summary>
    <h3>More information</h3>
  </summary>
  <div>
    <p>Additional details that can be expanded...</p>
  </div>
</details>
```

## When to Use

- Feature announcements
- Inline CTAs
- Embedded widgets
- Contextual cards
- Tip/note boxes
- Inline forms
