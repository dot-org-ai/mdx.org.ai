# Modal Container

A centered dialog overlay using the native `<dialog>` element for built-in accessibility and keyboard handling.

<Preview>
  <dialog open>
    <article>
      <header>
        <h2>Confirm Action</h2>
        <button aria-label="Close">×</button>
      </header>
      <p>Are you sure you want to continue? This action cannot be undone.</p>
      <footer>
        <menu>
          <li><button>Cancel</button></li>
          <li><button>Confirm</button></li>
        </menu>
      </footer>
    </article>
  </dialog>
</Preview>

## Semantic Pattern

Use the native `<dialog>` element:

```html
<dialog id="confirm">
  <article>
    <header>
      <h2>Confirm Action</h2>
      <button onclick="this.closest('dialog').close()">×</button>
    </header>
    <p>Are you sure you want to continue?</p>
    <footer>
      <menu>
        <li><button onclick="this.closest('dialog').close()">Cancel</button></li>
        <li><button onclick="handleConfirm()">Confirm</button></li>
      </menu>
    </footer>
  </article>
</dialog>

<!-- Open the dialog -->
<button onclick="document.getElementById('confirm').showModal()">
  Delete Item
</button>
```

## Why Native `<dialog>`?

The `<dialog>` element provides:
- **Built-in accessibility** - Focus management and screen reader support
- **Keyboard handling** - Escape key closes the dialog
- **`::backdrop`** - Native backdrop styling
- **`showModal()`** - Traps focus and adds backdrop
- **`close()`** - Closes dialog and restores focus
- **No JavaScript required** - Works with `open` attribute for non-modal

## JavaScript

```js
const dialog = document.getElementById('confirm')

// Open as modal (with backdrop)
dialog.showModal()

// Open as non-modal (no backdrop)
dialog.show()

// Close
dialog.close()

// Close with return value
dialog.close('confirmed')

// Handle close event
dialog.addEventListener('close', () => {
  console.log(dialog.returnValue) // 'confirmed' or ''
})

// Close on backdrop click
dialog.addEventListener('click', (e) => {
  if (e.target === dialog) {
    dialog.close()
  }
})
```

## Variations

### With Form

Use `method="dialog"` to close on submit:

```html
<dialog id="edit">
  <form method="dialog">
    <header>
      <h2>Edit Profile</h2>
    </header>
    <fieldset>
      <label>Name <input type="text" name="name" required></label>
      <label>Email <input type="email" name="email" required></label>
    </fieldset>
    <footer>
      <menu>
        <li><button type="button" onclick="this.closest('dialog').close()">Cancel</button></li>
        <li><button type="submit">Save</button></li>
      </menu>
    </footer>
  </form>
</dialog>
```

## When to Use

- Confirmation dialogs
- Form submissions
- Alert messages
- Focused tasks
