# Docs Layout

A three-column layout for documentation with sidebar navigation, main content, and table of contents.

<Preview>
  <article>
    <aside>
      <nav aria-label="Main">
        <section>
          <h4>Getting Started</h4>
          <menu>
            <li><a href="/intro">Introduction</a></li>
            <li><a href="/install" aria-current="page">Installation</a></li>
            <li><a href="/quickstart">Quick Start</a></li>
          </menu>
        </section>
      </nav>
    </aside>
    <main>
      <h1>Installation</h1>
      <p>Learn how to install and configure BuilderCSS.</p>
      <h2 id="npm">Using npm</h2>
      <pre><code>npm install buildercss</code></pre>
    </main>
    <aside>
      <nav aria-label="On this page">
        <h4>On this page</h4>
        <menu>
          <li><a href="#npm">Using npm</a></li>
          <li><a href="#cdn">Using CDN</a></li>
        </menu>
      </nav>
    </aside>
  </article>
</Preview>

## Semantic Pattern

A docs layout is an `<article>` with two `<aside>` elements flanking `<main>`:

```html
<article>
  <aside>
    <nav aria-label="Main">
      <section>
        <h4>Getting Started</h4>
        <menu>
          <li><a href="/intro">Introduction</a></li>
          <li><a href="/install" aria-current="page">Installation</a></li>
        </menu>
      </section>
    </nav>
  </aside>
  <main>
    <!-- Documentation content -->
  </main>
  <aside>
    <nav aria-label="On this page">
      <h4>On this page</h4>
      <menu>
        <li><a href="#overview">Overview</a></li>
        <li><a href="#examples">Examples</a></li>
      </menu>
    </nav>
  </aside>
</article>
```

## CSS

```css
/* Match article with aside + main + aside */
article:has(> aside + main + aside) {
  display: grid;
  grid-template-columns: 280px 1fr 200px;
  min-height: 100vh;
}

article:has(> aside + main + aside) > aside:first-child {
  background: var(--muted);
  border-right: 1px solid var(--border);
  padding: var(--spacing-4);
  position: sticky;
  top: 0;
  height: 100vh;
  overflow-y: auto;
}

article:has(> aside + main + aside) > main {
  padding: var(--spacing-8);
  max-width: 800px;
}

article:has(> aside + main + aside) > aside:last-child {
  border-left: 1px solid var(--border);
  padding: var(--spacing-4);
  position: sticky;
  top: 0;
  height: 100vh;
}

article:has(> aside + main + aside) aside h4 {
  font-size: var(--text-xs);
  font-weight: 600;
  text-transform: uppercase;
  margin-bottom: var(--spacing-2);
}

article:has(> aside + main + aside) aside menu {
  list-style: none;
  padding: 0;
  font-size: var(--text-sm);
}

article:has(> aside + main + aside) aside a[aria-current] {
  color: var(--primary);
  font-weight: 500;
}

@media (max-width: 1024px) {
  article:has(> aside + main + aside) {
    grid-template-columns: 280px 1fr;
  }

  article:has(> aside + main + aside) > aside:last-child {
    display: none;
  }
}

@media (max-width: 768px) {
  article:has(> aside + main + aside) {
    grid-template-columns: 1fr;
  }

  article:has(> aside + main + aside) > aside:first-child {
    display: none;
  }
}
```
