## Semantic Pattern

A FAQ section uses `<details>` elements inside a `<dl>` (definition list):

```html
<section>
  <header>
    <h2>Frequently Asked Questions</h2>
    <p>Everything you need to know.</p>
  </header>
  <dl>
    <details>
      <summary>Question one?</summary>
      <p>Answer one.</p>
    </details>
    <details>
      <summary>Question two?</summary>
      <p>Answer two.</p>
    </details>
    <details open>
      <summary>Question three?</summary>
      <p>Answer three (open by default).</p>
    </details>
  </dl>
</section>
```

## Why `<details>` Instead of Buttons?

The `<details>` element provides:
- **Native accessibility** - Keyboard navigation and screen reader support built-in
- **No JavaScript required** - Open/close works without JS
- **Semantic meaning** - Browsers understand this is disclosure content
- **The `open` attribute** - Easy styling for open/closed states

## When to Use

- FAQ pages
- Product questions
- Support pages
- Documentation
