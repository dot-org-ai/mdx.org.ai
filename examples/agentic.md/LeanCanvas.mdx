# Lean Canvas: agentic.md

## Problem

1. **Complexity Barrier** - Building AI agents requires deep knowledge of frameworks, LangChain, LlamaIndex, etc.
2. **Iteration Speed** - Testing and iterating on agent behavior is slow and cumbersome
3. **Deployment Friction** - Going from prototype to production agent is painful

### Existing Alternatives
- LangChain (complex, Python-focused)
- LlamaIndex (data-focused, steep learning curve)
- Custom implementations (slow, expensive)
- No-code builders (limited, vendor lock-in)

## Customer Segments

### Early Adopters
- **Solo developers** building AI-powered side projects
- **Startup CTOs** needing to ship agent features fast
- **AI hobbyists** experimenting with agent concepts

### Mainstream
- **Development teams** at SaaS companies adding AI
- **Agencies** building AI solutions for clients
- **Enterprise developers** with agent requirements

"The markdown-first agent framework"

## Solution

1. **Markdown Agent Definition** - Define agents in `.mdx` files with frontmatter
2. **Built-in Tool Library** - Pre-built integrations for common services
3. **One-Command Deploy** - `npx agentic deploy` ships to production
4. **MCP Server** - Agents work with Claude Desktop out of the box
