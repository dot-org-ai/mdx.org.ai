# Lean Canvas: readme.mdx.org.ai

## Problem

1. **Documentation Neglect** - READMEs are always out of date or incomplete
2. **Time Constraints** - Developers prioritize code over docs
3. **First Impression Loss** - Poor READMEs hurt project adoption

### Existing Alternatives
- Manual writing (time-consuming)
- README templates (generic)
- ChatGPT (copy/paste, no analysis)
- readme.so (templates only)

## Customer Segments

### Early Adopters
- **Open source maintainers** with many projects
- **Solo developers** shipping fast
- **Developer advocates** maintaining examples

### Mainstream
- **Engineering teams** with internal tools
- **Startups** with developer-facing products
- **Agencies** delivering code to clients

## Unique Value Proposition

**Great Docs Without the Effort**

AI analyzes your codebase and generates comprehensive READMEs. Always accurate, always current.

### High-Level Concept
"AI README generation"

## Solution

1. **Code Analysis** - AI reads your codebase
2. **README Generation** - Installation, usage, API docs
3. **Auto-Updates** - README syncs when code changes
4. **GitHub Integration** - Works in your workflow
