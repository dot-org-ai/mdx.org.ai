# Lean Canvas: writer.md

## Problem

1. **Tool Switching** - Jumping between editor and AI breaks flow
2. **Format Conversion** - AI tools output that needs reformatting
3. **Workflow Mismatch** - Web AI tools don't fit terminal-based work

### Existing Alternatives
- ChatGPT (web-based, copy/paste)
- GitHub Copilot (code-focused, not prose)
- Jasper (marketing-focused, not technical)
- Manual writing (slow, effort-intensive)

## Customer Segments

### Early Adopters
- **Developers** writing documentation
- **Technical writers** in terminal environments
- **Open source maintainers** updating READMEs

### Mainstream
- **Documentation teams** at tech companies
- **DevRel professionals** creating content
- **Content teams** supporting developers

## Unique Value Proposition

**AI Writing, Native to Your Workflow**

Stay in the terminal. Get AI help. Markdown in, markdown out.

### High-Level Concept
"CLI AI writing assistant"

## Solution

1. **CLI Interface** - Generate and edit from terminal
2. **Markdown Native** - Input and output in markdown
3. **Git Integration** - Works with version control
4. **Editor Plugins** - VS Code, Vim, Emacs support
