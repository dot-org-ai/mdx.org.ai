# Code Reviewer

The Code Reviewer agent provides thorough, actionable code reviews for every pull request. It catches bugs, security issues, and style violations before they reach production.

## Usage

```typescript
import { CodeReviewer } from '@agents/code-reviewer'

const reviewer = new CodeReviewer({
  model: 'claude-3-opus',
  github: githubClient,
  styleGuide: './docs/style-guide.md',
  securityRules: 'owasp-top-10',
})

// Review a PR
const review = await reviewer.review({
  repo: 'myorg/myapp',
  pr: 123,
})

// Get specific analysis
const security = await reviewer.checkSecurity(diff)
const performance = await reviewer.checkPerformance(diff)
```

## GitHub Action

```yaml
# .github/workflows/review.yml
name: AI Code Review
on: [pull_request]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: agents/code-reviewer@v2
        with:
          github-token: ${{ secrets.GITHUB_TOKEN }}
          model: claude-3-sonnet
          style-guide: ./docs/style-guide.md
```

## Review Categories

### Security
- SQL injection vulnerabilities
- XSS attack vectors
- Authentication bypasses
- Sensitive data exposure
- Insecure dependencies

### Bugs
- Logic errors
- Null pointer dereferences
- Race conditions
- Memory leaks
- Incorrect error handling

### Style
- Naming conventions
- Code formatting
- Documentation requirements
- File organization
- Import ordering

### Performance
- N+1 queries
- Unnecessary re-renders
- Memory-intensive operations
- Missing caching opportunities

## Configuration

```yaml
# .code-reviewer.yml
severity:
  security: critical
  bugs: high
  style: low
  performance: medium

ignore:
  - "**/*.test.ts"
  - "**/fixtures/**"

customRules:
  - name: no-console-log
    pattern: "console.log"
    message: "Use logger instead of console.log"
    severity: warning
```
