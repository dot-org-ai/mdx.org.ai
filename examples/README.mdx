# MDX Examples

Build AI-powered applications with Markdown. Each example is a working product you can run, customize, and deploy.

MDXLD uses a URL-based file system where every resource can be both a file AND a folder—unlike traditional file systems. The folder name is the domain (e.g., `assistant.md/`), and everything is MDX, combining data, content, code, and UI in a single format.

For simple projects, `README.mdx` serves as both the `/` endpoint and documentation. When you need them separate, add `index.mdx` for the public website while `README.mdx` remains the technical docs (like GitHub).

## Quick Start

```bash
# Clone and install
git clone https://github.com/mdx-org/mdx.org.ai
cd mdx.org.ai
pnpm install

# Run any example
pnpm mdxe dev examples/assistant.md
```

## Structure

Each example folder IS the domain. `README.mdx` is the `/` endpoint (or `index.mdx` if you need separate website vs docs):

```
domain.tld/              # Folder name = domain = namespace
├── README.mdx           # Technical docs (/ endpoint if no index.mdx)
├── index.mdx            # Website/landing page (/ endpoint if present)
├── app.mdx              # Application layout
├── api.mdx              # API reference
├── mcp.mdx              # MCP server config
├── [Type].mdx           # Dynamic route templates (e.g., /users/{id})
└── docs/                # Nested routes (/docs/*)
    └── index.mdx        # /docs endpoint
```

## Running Examples

```bash
pnpm mdxe build examples/assistant.md

# Deploy to Cloudflare
pnpm mdxe deploy examples/assistant.md
```

## Creating New Examples

```bash
# Create folder named after your domain
mkdir examples/my-product.dev
cd examples/my-product.dev

# Create README.mdx as the / endpoint
cat > README.mdx << 'EOF'
---
$type: https://mdx.org.ai/Package
$id: https://my-product.dev
$context: https://my-product.dev
name: my-product.dev
version: 0.1.0
---

export const config = {
  // Configuration
}

export async function main() {
  // Implementation
}

# my-product.dev

Description and documentation...
EOF
```
