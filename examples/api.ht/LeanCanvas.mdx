# Lean Canvas: api.ht

## Problem

1. **Documentation Gap** - APIs require separate documentation that's always out of sync
2. **Discoverability** - Finding the right endpoint requires digging through docs
3. **Developer Experience** - REST APIs don't self-describe; you need to read docs first

### Existing Alternatives
- Swagger/OpenAPI (still requires separate docs)
- GraphQL (complex, different paradigm)
- Manual documentation (always stale)
- Postman collections (not integrated with API)

## Unique Value Proposition

**APIs You Can Click**

### High-Level Concept
"Hypertext APIs that browse like websites"

## Solution

1. **Self-Describing Endpoints** - Every URL returns HTML in browser, JSON for machines
2. **Clickable Navigation** - Browse your API like a website, follow links naturally
3. **Inline Documentation** - Docs are part of the response, not separate files
4. **Schema Autocompletion** - Clients get autocomplete from API responses
